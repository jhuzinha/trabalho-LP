01 A linguagem · o projeto · a comparação

Concorrente em Go

Consultar 10 lojas ao mesmo tempo em vez de uma por vez — usando os três construtores que definem a linguagem: goroutines, channels e select. Um projeto pequeno que exercita o que Go faz de melhor: concorrência simples, legível e confiável.

02 A linguagem

Linguagem: histórico e versão

  • 2007 — concebida no Google por Robert Griesemer, Rob Pike e Ken Thompson (Pike e Thompson vêm do Unix e do C).
  • Nov/2009 — anunciada como projeto open source (licença BSD, gratuita).
  • Mar/2012 — Go 1.0 e a "promessa de compatibilidade": código de 2012 ainda compila hoje.
  • Hoje — Go 1.24 (2025). Compilada, tipagem estática, garbage collector, sintaxe da família C.

A motivação foi concreta: builds de C++ lentíssimos, dependências emaranhadas e concorrência difícil. A resposta foi uma linguagem minimalista — só 25 palavras-chave.

# infraestrutura moderna é escrita em Go Docker contêineres Kubernetes orquestração Terraform infra as code Prometheus observabilidade Grafana dashboards # filosofia da linguagem "Less is exponentially more." — Rob Pike
03 O projeto

Projeto: premissas, usuário e domínio

Domínio

Web scraping concorrente

Coleta de preços de um mesmo produto em várias lojas via HTTP, decodificando JSON e montando um ranking.

Usuário

Consumidor & estudante

Quem quer o menor preço sem abrir 10 abas — e quem quer ver concorrência funcionando de forma visual no terminal.

Premissa central

Rede é espera, não CPU

O gargalo é o tempo de resposta da rede. Logo, disparar N requisições simultâneas reduz o tempo total ao da loja mais lenta.

Módulo 1

webscraper (mock)

Servidor HTTP local simula 10 lojas com latências distintas — demo determinística, sem depender da internet.

Módulo 2

webscraper-real

Consulta a API pública dummyjson.com de verdade — dados reais, preços com desconto e avaliação.

Módulo 3

comparacao.py

Mesmo problema em Python (sequencial × threads × pool) para medir Go contra outra linguagem.

04 Critério de avaliação

Construtores

  • go f() — dispara uma goroutine: função rodando concorrentemente. Custa ~2 KB de pilha inicial.
  • chan T — channel tipado: o "cano" por onde goroutines trocam dados com segurança, sem locks manuais.
  • <-ch — envio/recepção que sincroniza: recebe bloqueia até haver dado.
  • select — espera vários canais ao mesmo tempo; o primeiro pronto vence. É o coração do timeout.
  • struct · interface · defer · slice · map — mais o resto do vocabulário do dia a dia.
Go — palavra-chave da linguagem
// uma goroutine por loja — ~2KB cada for _, s := range Stores { go scrapePrice(s.Name, url, ch) } r := <-ch // channel sincroniza, sem mutex select { // timeout nativo case r := <-done: ch <- r case <-time.After(t): ch <- Result{TimedOut: true} }
C — biblioteca + sincronização à mão
#include <pthread.h> /* threads do SO, ~1MB cada */ pthread_t th[N]; for (int i = 0; i < N; i++) pthread_create(&th[i], NULL, scrape, &a[i]); for (int i = 0; i < N; i++) pthread_join(th[i], NULL); pthread_mutex_lock(&mtx); /* proteger à mão */ results[n++] = r; pthread_mutex_unlock(&mtx); /* timeout? poll()/select() em fds — bem mais trabalho */

Em Go, concorrência são 3 palavras-chave — go, chan, select. Em C, é biblioteca (pthreads) mais mutex e timeout montados manualmente.

05 Critério de avaliação

Legibilidade

  • gofmt — formatador oficial embutido. Não há debate sobre estilo: existe uma forma canônica e a ferramenta aplica.
  • 25 palavras-chave — a especificação inteira cabe na cabeça. Pouca sintaxe para decorar.
  • Erros explícitos — if err != nil deixa o caminho de falha visível na leitura, sem exceções escondidas.
  • Sem pré-processador, sem headers, sem macros — o que você lê é o que executa.

A prova é lado a lado: a mesma busca HTTP + leitura de JSON nas duas linguagens.

Go — net/http + encoding/json (padrão)
resp, err := http.Get(url) if err != nil { return err } defer resp.Body.Close() json.NewDecoder(resp.Body).Decode(&data)
C — libcurl + parser JSON externo
#include <curl/curl.h> /* dependência externa */ CURL *c = curl_easy_init(); curl_easy_setopt(c, CURLOPT_URL, url); curl_easy_setopt(c, CURLOPT_WRITEFUNCTION, cb); if (curl_easy_perform(c) != CURLE_OK) { ... } curl_easy_cleanup(c); /* + cJSON à parte só para LER o JSON */

Além disso, em C a legibilidade sofre com #include/#define, aritmética de ponteiros e estilos divergentes entre projetos — Go tem gofmt, um só jeito.

06 Critério de avaliação

Capacidade de escrita

net/http

Cliente HTTP embutido

Um http.Get(url) e pronto. Em C seria libcurl (dependência externa) + configuração manual.

encoding/json

JSON → struct direto

Decodifica para tipos com tags declarativas. Em C não há JSON nativo — precisa de cJSON/jansson.

go · chan · select

Concorrência de 1ª classe

Sem pthread_create, mutex e join manuais como em C.

GOOS=windows go build

Cross-compilation trivial

Comprovado neste teste: o mesmo código gerou binário Windows a partir do Linux, com um comando.

bash — feito durante o teste deste projeto
$ go build -o ws-real . ✓ BUILD real OK $ GOOS=windows go build -o ws-real.exe . ✓ cross-build windows OK
07 Critério de avaliação

Confiabilidade

  • Garbage collector — sem malloc/free. Adeus use-after-free, double-free e vazamentos manuais.
  • Sem aritmética de ponteiro insegura — nada de estourar buffer por acidente como em C.
  • Tipagem estática forte — erros de tipo pegos em tempo de compilação.
  • Race detector — go run -race encontra condições de corrida entre goroutines.
  • Canal com buffer 1 — no projeto, make(chan Result, 1) evita goroutine leak se o timeout vencer (documentado no código).

09 Código do projeto

Exemplos

webscraper/main.go — o fluxo completo
// 1. canal com buffer: goroutines enviam sem travar umas às outras ch := make(chan Result, len(mock.Stores)) // 2. dispara TODAS as lojas ao mesmo tempo — uma goroutine cada for _, store := range mock.Stores { url := fmt.Sprintf("http://localhost:%d%s", serverPort, store.Path) go scrapePrice(store.Name, url, ch) } // 3. coleta pelo canal — a ordem é a VELOCIDADE da loja, não a da lista for range mock.Stores { r := <-ch // bloqueia até QUALQUER goroutine enviar results = append(results, r) printResult(r) }

Três linhas de conceito — disparar, esperar, coletar — resolvem concorrência que em C exigiria dezenas de linhas de pthreads e sincronização manual.

14 Conclusão

Go, em resumo

Go pega a sintaxe familiar do C e troca gerência manual de memória e concorrência frágil por goroutines, channels e um GC. O projeto prova isso na prática: 6.3× mais rápido que serial, 20.7× mais que Python sequencial — em código que cabe numa tela.
